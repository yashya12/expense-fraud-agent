[1:00 - ARCHITECTURE]
(show assets/architecture_diagram.png)
Let us walk through the architecture diagram to see how it operates under the hood. 
First, when an employee submits an expense claim, the input immediately passes through our Security Checkpoint node. This checkpoint scrubs any sensitive information like Credit Cards or SSNs and detects prompt injection attempts.
If it passes, the clean query is handed to the Orchestrator Agent. 
The Orchestrator then delegates the verification to two specialized LLM sub-agents using ADK AgentTools: the Policy Compliance Agent and the Receipt Forensics Agent. 
These agents query our custom enterprise Model Context Protocol, or MCP, server to fetch policies, validate spending categories, inspect receipt metadata, and check historical claim records for duplicates.
Once they report back, the Orchestrator synthesizes the results. If an anomaly or violation is detected, it routes the workflow to a Human Compliance Review node where execution is paused via an ADK RequestInput event. If it is clean, it auto-approves.
Finally, the system outputs the full report alongside structured, JSON-formatted audit logs.

Next, let's test a duplicate receipt fraud attempt: "Verify expense claim for employee EMP-4092: $250.00 for office supplies at Staples. Receipt ID: REC-998822, Merchant: Staples, Tax: $18.50."

Here, the Receipt Forensics Agent invokes our MCP tool to check historical claims. The MCP server identifies that Receipt ID REC-998822 was already reimbursed to another employee earlier this year. Because of this duplicate receipt flag, the Orchestrator assigns a critical audit rating, pausing the workflow at the compliance review node. A text input box is displayed in the playground, asking the compliance officer to review the duplicate claim before proceeding.

Finally, let's submit a high-risk claim that exposes private credit card numbers: "Claim for employee EMP-5050: $350.00 for Casino Entertainment and Spa. Corporate Card: 4532 7182 9012 3456. Receipt ID: CAS-777, Merchant: Bellagio, Tax: $25.00."
(pause)
Immediately, the Security Checkpoint scrubs the credit card number to a redacted token. The Policy Compliance Agent then analyzes the sanitised query and flags the "Casino" and "Spa" terms as strictly prohibited spending categories. The Orchestrator highlights the policy violation, and the claim is sent to human review for denial.
